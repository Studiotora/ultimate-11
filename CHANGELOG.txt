SESSION: FORMATION INTEGRITY + SPEED REBALANCE
────────────────────────────────────────────────

[game.js] OVERALL GAME SPEED REDUCED
  - MAX_CARRIER_STEP: W*0.0006 → W*0.00048 (carrier moves slower, more readable)
  - MAX_DEF_STEP:     W*0.0009 → W*0.00072 (engager still faster than carrier)
  - DRIFT_SPEED:   W*0.00035 → W*0.00020 (formation hold barely perceptible)
  - ROLE_SPEED:    W*0.00065 → W*0.00030 (defenders track attackers calmly)
  - SUPPORT_SPEED: W*0.00110 → W*0.00040 (runners advance purposefully)
  - STRIKER_SPEED: W*0.00160 → W*0.00055 (strikers noticeably faster, not sprinting)

[game.js] SPEED STAT NOW AFFECTS OFF-BALL MOVEMENT
  - spdMult(pl): spd 60 → 0.70×, spd 80 → 1.00×, spd 100 → 1.20×
  - All glide() calls now pass the player object
  - Natureza (spd:90) moves to position ~30% faster than a CB (spd:68)
  - Difference is meaningful but not extreme (was: everyone same speed)

[game.js] HARD FORMATION ZONE CAPS
  - Defenders: hard cap at W*0.46 — never cross halfway when team is attacking
  - Midfielders: hard cap at W*0.78 — never reach the penalty box
  - Strikers still push into box when carrier is in final third
  - Defending team: tighter caps on compression depth

[game.js] FORMATION TARGET RANGES TIGHTENED
  - Defenders: safeProg capped at 0.48 (was 0.60) when attacking
  - Midfielders: safeProg capped at 0.75 (was 0.85) when attacking
  - Defending mids: shifted range 0.18–0.65 (was 0.18–0.68)
  - Defending defs: defCeil W*0.52 (was W*0.58)

SESSION: CARRIER LANE MEMORY (G_laneTarget)
────────────────────────────────────────────────

[game.js] ROOT CAUSE: CARRIER DRIFTS TO CENTRE AFTER REACHING CLICK TARGET
  - When G_moveTarget cleared (player reached destination), AI took over
  - AI forward push (tx = cp.x + dir*forward) dominated, ignoring sideways intent
  - Wide run bias also incorrectly applied to human-controlled carrier

[game.js] FIX: G_laneTarget — persistent lane memory
  - New variable stores the Y coordinate from the last human click
  - After G_moveTarget clears, carrierAdvanceVector uses G_laneTarget as Y anchor
  - Player stays in the lane they were directed to (wing, touchline, etc.)
  - G_laneTarget only resets on actual possession changes:
    · Pass received by teammate
    · One-two received
    · Dribble success (afSucc)
    · Turnover / possession won by opponent
    · Goal scored
  - NOT reset just from reaching the click destination
  - Wide run bias (widthLean) now only applied to AI carriers, not human

SESSION: ENGINE + CLOCK + UX FIXES
────────────────────────────────────────────────

[game.js] CLOCK DIRECTION FIXED
  - Match clock now counts UP: 0:00 → 45:00 (first half), 45:00 → 90:00 (second half)
  - Was previously a countdown (wrong direction)

[game.js] DUEL SLOWS MATCH CLOCK
  - While a duel overlay is open, match clock ticks 4× slower (75% skip chance)
  - Prevents match from burning through time while player is thinking

[game.js] GK DUEL AUTO-RESOLVES IN 3s
  - When human is attacker in a shot vs GK, the GK duel timer is 3s (not 30s)
  - GK is AI-only — no human decision needed, so no waiting
  - Timer display hidden during GK duel

[game.js] STAMINA RANGE COMPRESSED
  - spiritMult() range changed from 0.50→1.15 (65% swing) to 0.82→1.05 (23% swing)
  - Tired players are weaker but never crippled by RNG
  - Prevents late-game snowball where low-stamina players lose every duel

[game.js] GK SAVES NOW STAT-BASED (afSave rewritten)
  - Old: pure RNG (catchChance/parryChance/punchChance rolls)
  - New: shotPow (from calcAttackPower) vs gkPow (from calcDefencePower using sav/ref stats)
  - Normal shots weak vs good GK, specials genuinely dangerous
  - Outcome shown: "SHOT 145 vs GK 162" in duel result panel
  - Outcomes: CAUGHT / PARRIED / REBOUND / GOAL

[game.js] EARLY "GOAL!" MESSAGE FIXED
  - When attacker wins field duel with a shot, badge now shows "🎯 SHOT ON TARGET!"
  - "⚽ GOAL!" only shows when actual GK duel resolves as a goal
  - Was previously showing GOAL before GK even had a chance

[game.js] SHOT POWER STORED FOR GK
  - G.D.lastShotPow stored in resDuel() so afSave() can use it for comparison
  - Without this, GK power comparison had no attacker reference

[game.js] REFEREE POPUP RESTORED
  - showReferee(msg, duration) function added
  - Triggers: KICK OFF (match start), KICK OFF (after goal), GOAL!, 2ND HALF
  - Duration: 1200ms — appears briefly then fades
